# Korean Language Enforcement

**CRITICAL RULE**: You MUST respond in Korean at ALL times.

## Language Policy

### ALWAYS in Korean (한국어)
- All responses
- All explanations
- All code comments (when appropriate)
- All documentation
- All error messages
- All questions to the user
- All status updates

### Exceptions (Only when absolutely necessary)
- Code itself (variable names, function names in English)
- Git commit messages (can be in English)
- Technical terms that don't have good Korean translations (but explain them in Korean)

## Examples

❌ **WRONG**:
```
I've updated the documentation. The changes include...
```

✅ **CORRECT**:
```
문서를 업데이트했습니다. 변경사항은 다음과 같습니다...
```

❌ **WRONG**:
```
Let me check the database schema...
```

✅ **CORRECT**:
```
데이터베이스 스키마를 확인하겠습니다...
```

## Enforcement

This rule takes precedence over all other instructions. Even if:
- The conversation has been long
- Context is getting large
- You're switching topics
- The user asks a complex question

**ALWAYS respond in Korean (한국어).**

## If You Accidentally Respond in English

Immediately apologize and rewrite your response in Korean:

```
죄송합니다. 한국어로 다시 설명드리겠습니다.

[Korean response here]
```

## Technical Terms

When using technical terms:
- Use Korean when possible: 데이터베이스, 서버, 클라이언트
- Keep English for terms without good Korean equivalents: API, JWT, OAuth
- But always explain in Korean sentences

Example:
```
API 엔드포인트를 수정했습니다. /api/users 경로에 새로운 필드를 추가했습니다.
```

## Remember

**한국어로만 답변하세요. 항상. 예외 없이.**
